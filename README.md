# Мультиагентный ассистент авиакомпании (Flight Agent System)

Этот проект представляет собой мультиагентную систему (Multi-Agent System) для поддержки клиентов авиакомпании, созданную с использованием фреймворка [LangGraph](https://github.com/langchain-ai/langgraph). Система построена на базе паттерна Coordinator-Specialist, где главный агент-координатор распределяет задачи между специализированными агентами. А также включает CriticAgent для контроля качества генерируемых ответов.

## Основные возможности

- **Агент-координатор**: Принимает запрос пользователя, декомпозирует его на подзадачи и делегирует их специалистам.
- **Специализированные агенты**:
  - `FlightAgent` — ищет рейсы и предоставляет детали по ним.
  - `PolicyAgent` — консультирует по правилам авиакомпании (багаж, возврат, отмена).
  - `BookingAgent` — предоставляет информацию о бронированиях и позволяет вносить изменения (например, перебронировать рейс).
- **Critic Agent (Reflexion pattern)**: Проверяет итоговый ответ координатора на полноту, точность, безопасность и вежливость перед отправкой пользователю. Если ответ не проходит проверку, координатор переделывает его.
- **Долгосрочная память (Long-Term Memory)**: Система запоминает предпочтения пользователя (например, имя, предпочтения в еде, любимые места) и использует их в последующих запросах.
- **Краткосрочная память (Short-Term Memory)**: Агенты помнят контекст текущей сессии (диалога).

## Установка и запуск

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Alexander8841/airline-ai-system.git
   cd airline-ai-system
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Настройте ключи API:
   Скопируйте пример конфигурации и укажите свой API ключ в `.env` файле:
   ```bash
   cp .env.example .env
   ```
   Внутри `.env` файла пропишите `GOOGLE_API_KEY` (или ключи Yandex, если используете YandexGPT).

4. Запустите Streamlit-приложение:
   ```bash
   streamlit run app.py
   ```

## Структура проекта

- `app.py` — веб-интерфейс на базе Streamlit и управление состоянием.
- `src/`
  - `agents.py` — логика агентов: `SpecializedAgent`, `EnhancedCoordinator`, `CriticAgent`.
  - `config.py` — системные промпты и настройка LLM.
  - `database.py` — заглушка базы данных (загрузка данных рейсов, бронирований, политик и профиля пассажира).
  - `graph.py` — логика графа исполнения (StateGraph) LangGraph для ReAct и MAS.
  - `tools.py` — инструменты (tools), доступные агентам (поиск, информация, бронирование, обновление профиля).
  - `evaluation.py` — сценарии и метрики для тестирования качества работы агентов.
- `data/` — JSON-файлы с моковыми данными базы данных.

## Результаты оценки (Evaluation)

В рамках проекта было проведено автоматическое сравнительное тестирование двух архитектур (Single-Agent ReAct и Multi-Agent System) на наборе из 27 задач различной сложности. Оценка качества производилась по подходу **LLM-as-a-Judge** с замером полезности (Usefulness), фактической точности (Groundedness) и эффективности использования инструментов (Efficiency).

**Итоговая таблица (Scoreboard):**

| Архитектура | Средний балл (из 6) | Согласованность с эталоном (Cohen's Kappa) | Вызовы инструментов | Затраты |
| :--- | :---: | :---: | :---: | :---: |
| **Human GT (Эталон)** | 5.07 | 1.000 | 0 | $0.000 |
| **ReAct v2 (Обычный)** | 4.85 | 0.111 | 57 | $0.01425 |
| **Multi-Agent (MAS)** | **5.07** | **0.180** | **27** | **$0.00675** |

**Ключевые выводы:**
1. **Эффективность (Cost & Tools):** MAS оказался в 2 раза дешевле и эффективнее обычного ReAct. Благодаря строгому разделению обязанностей оркестратор перестал путаться и зацикливаться — количество вызовов API сократилось с 57 до 27.
2. **Точность ответов (Quality):** Средний балл качества MAS (5.07) идеально совпал с эталонной человеческой оценкой, а уровень согласованности (Kappa) вырос почти на 70%.
3. **Безопасность (Critic):** Добавление рефлексии (Critic Agent) полностью исключило галлюцинации и придумывание "из головы" несуществующих рейсов или цен.
